Notes

  1. The features of the apps, as well as their security and privacy details shown in the table, are conditional on the ease of access to this information (on April 12).

  2. If an aplication has 2 versions of its product and one of them is healthcare-based, the healthcare-based was analyzed.

  3. The features of each health-based platform were gathered for the most complete version (e.g. If there are three pricing plans for an application, the features of the most complete one was selected).

  4. "No *" in [Security standards] column means that we have not found information about it. Obviously, no any application says explicitly that they are not compliant with security standards. Nevertheless, all of them should be GDPR compliant to be legally used. HIPAA and SOC2 (and others) are aditional security standards.

  5. We only state GDPR in the [Security standards] column when there was an official document from the application stating GDPR compliance.
